# 文檔維護指引

用於在每次功能或行為變動時，快速確認需要同步更新的文件與 README 撰寫方向，避免資訊不一致。

## 變更分類
- 功能新增/修改：新參數、新按鈕、新流程。
- 行為/相容性變更：預設值、輸出格式、依賴版本。
- UI/UX 調整：版面、操作步驟、提示訊息。
- 破壞性變更：移除功能、輸出不向後相容、需要使用者手動處理。

## 更新清單（依變更類型勾選）
- 摘要（短版）：`docs/changelog.md` — 列出使用者能感知的變動。
- 詳細變更：`UPDATE_LOG.md` — 描述背景、邏輯與細節。
- CLI 說明：`docs/cli.md`、`USAGE.md` — 參數、範例、預設值。
- GUI 說明：`docs/gui.md` — 操作步驟、選項效果、預覽/輸出行為。
- 精準/性能說明：`docs/precision.md` — 精準模式、硬體加速、副作用。
- 規格書：`README.md`（主規格書）、`UI_SPEC.md`（UI 規格） — 需求/限制若有變更需同步。
- 環境需求：`docs/setup.md`、`requirements.txt` — 新增依賴或安裝步驟變更。
- 版本標記：在 `UPDATE_LOG.md`/`docs/changelog.md` 加上版本標題（如 v5）與日期；標註是否為破壞性變更。

## 提交前檢查表
1) 變更點有無對應到以上文件？（至少摘要 + 詳細 + 對應的 CLI/GUI/規格）
2) README/規格書的描述是否符合現行行為與限制？
3) 範例與參數預設值已更新且可執行？
4) 若為破壞性變更，是否在摘要與 README/USAGE 中清楚標註？

## README 撰寫方向（範本）
- 專案定位：一句話說明目標與適用情境。
- 功能重點：使用者能得到什麼（快速裁切、字幕對齊、預覽等），避免實作細節。
- 安裝/需求：Python 版本、ffmpeg/ffprobe、PyQt；安裝指令。
- 快速開始：CLI 範例、GUI 啟動命令；最小工作流程。
- 主要參數/選項：僅列常用與會影響結果的選項；指向 `docs/cli.md`/`docs/gui.md` 取得完整說明。
- 精準 vs 快速：何時使用、輸出差異、性能/品質取捨（可連到 `docs/precision.md`）。
- 限制與已知問題：輸出時間誤差、字幕格式限制、平臺相容性。
- 版本資訊：最近一版重點（連到 `docs/changelog.md` 或 `UPDATE_LOG.md`），以及破壞性變更提醒。

## 建議的變更溝通流程
1) 開發中：在 PR 描述列出「變更摘要 + 影響面 + 需更新的文件清單」。
2) 開發完成：更新對應文件並在 PR 中勾選檢查表；附上一個 CLI 或 GUI 範例指令/截圖。
3) 發佈前：確認 `docs/changelog.md` 與 `UPDATE_LOG.md` 版本標題一致；若重寫歷史或大幅變更，提醒使用者需要重新下載/重設。

## PR/任務模板（可貼到描述區）
```
變更摘要：
- [ ] 功能
- [ ] 行為/預設值
- [ ] UI/UX
- [ ] 破壞性變更

文件同步：
- [ ] docs/changelog.md
- [ ] UPDATE_LOG.md
- [ ] docs/cli.md / USAGE.md
- [ ] docs/gui.md
- [ ] README.md / UI_SPEC.md
- [ ] docs/precision.md
- [ ] docs/setup.md / requirements

測試與範例：
- [ ] CLI 範例（命令/輸出）
- [ ] GUI 流程（若適用）
- [ ] 性能/精準模式確認（若適用）
```
